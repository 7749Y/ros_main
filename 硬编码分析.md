# ROS 项目硬编码问题分析

## 一、坐标/位姿硬编码

7 个节点的导航目标点与机械臂坐标全部写在 `.cpp` 源码中，没有任何外部配置：

| 文件 | 硬编码值 |
|------|---------|
| `arm_control.cpp` | 8 组机械臂坐标，如 `(150,0,120)`, `(107,185,42)`, `(107,128,45)` |
| `ar_tf_arm_5.cpp` | 导航目标 `(0.49384, 1.26898, 0, 1)`，机械臂观测点 `(100,170,120)` |
| `auto_charge.cpp` | 导航目标 `(0.3, 1.7499, 0.7, 0.7)` |
| `nav_goal.cpp` | 导航目标 `(2.18104548469018, 2.2049369564079457)` |
| `auto_bkhome.cpp` | 回撤目标 `(0.003, -0.001, -0.012, 1.000)` |
| `auto_nav_goal_4.cpp` / `auto_nav_goal_5.cpp` | 各一组导航目标 |
| `Grab.cpp` | 安全点、定位点、抓取点、放置点全部硬编码 |

**后果**: 换场地、换地图或重新标定后必须改源码重新编译。

---

## 二、AR 标签 ID 硬编码

```cpp
// ar_tf_arm_5.cpp
get_transform("Base", "ar_marker_1", ...)    // 标签 1 作为抓取目标
get_transform("Base", "ar_marker_100", ...)   // 标签 100 作为放置目标

// 所有含二次定位的文件
set_ARtrack(0, 0.4);   // 标签 ID=0，目标距离=0.4m

// Grab.cpp
// ID 100 → 定位标签；ID 1~4 → 货物标签
```

**后果**: 标签贴放位置变更或 ID 重规划必须改代码。

---

## 三、坐标参考系名硬编码

```cpp
goal.target_pose.header.frame_id = "map";   // 导航坐标系
srv.request.global_frame = "odom";           // 相对移动坐标系
listener.lookupTransform("Base", ..., ...);  // 机械臂基座坐标系
```

**后果**: 不同机器人使用 `base_link` / `base_footprint` 等不同命名规范时，复用成本极高。

---

## 四、ROS 话题/服务名硬编码

```cpp
"/goto_position", "/swiftpro/on", "/swiftpro/off",
"/relative_move", "/track", "hand_camera/ar_pose_marker"
```

**后果**: 命名空间变更或多机器人场景需要逐个修改所有源文件。应使用私有参数从 launch 传入。

---

## 五、时序/延时硬编码

```cpp
ros::Duration(1.0).sleep();
ros::Duration(2.0).sleep();
// Grab.cpp: 扫描超时写死 3 秒
```

**后果**: 机械臂型号、速度参数变更后延时不再可靠。应基于状态反馈而非固定 sleep。

---

## 六、机械臂偏移量硬编码

```cpp
// ar_tf_arm_5.cpp: grab() 默认偏移 10mm
void grab(x, y, z, off = 10) { ... }

// Grab.cpp: 货物上方偏移 40mm
float tag_z = cargo_z_surface + 40;
```

**后果**: 偏移量取决于货物大小、吸盘弹性、机械臂精度，应作为可调节参数。

---

## 七、区域编码硬编码

```cpp
setlocale(LC_CTYPE, "zh_CN.utf8");   // 7 个文件全部包含此行
```

**后果**: 国际化部署或非中文系统上可能异常。

---

## 八、节点名硬编码

```cpp
ros::init(argc, argv, "arm_control_node");
ros::init(argc, argv, "inbound_operation_node");
// 每个文件各自硬编码节点名
```

**后果**: 同一节点需启动多个实例时（如双机械臂）节点名冲突。应通过 launch 文件分配节点名。

---

## 总结

上述 8 类问题的根因相同：**参数与代码零分离**。所有业务数据（坐标、ID、延时、偏移）封死在 `.cpp` 源码中，而非通过 YAML 参数文件 + launch 文件注入。
