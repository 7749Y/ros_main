# ROS 项目架构分析

## 当前架构

```
ros_workspace/
├── src/                    → 5 包
│   ├── arm_control_pkg        (机械臂+吸盘控制)
│   ├── ar_tf_arm_5_pkg        (AR定位+导航+抓取完整流程)
│   ├── auto_bkhome_pkg        (返航 - 空壳)
│   ├── bringup/               (一键启动 launch)
│   └── state_machine_pkg      ★ 竞赛状态机框架
│
└── src_task_1/             → 7 包
    ├── auto_bkhome_pkg        (返航 - 实现)
    ├── auto_charge_pkg        ★ 从 src/ 移入
    ├── auto_nav_goal_4_pkg    (导航+AR定位)
    ├── auto_nav_goal_5_pkg    (导航+AR定位)
    ├── Grab_pkg               (视觉引导抓取)
    └── remove_back_pkg        ★ 新增：专用后撤节点
```

## 正面变化

- **state_machine_pkg**: 17 状态竞争状态机，参数从 launch 文件注入，支持超时保护
- **冗余清理**: `nav_goal_pkg`、`romove_pkg` 已删除；`auto_charge_pkg` 归并到 `src_task_1/`
- **remove_back_pkg**: 后撤动作独立为可执行节点，按需调用

## 仍存在的问题

### 代码零复用
核心函数在 6+ 个文件中重复实现：
```
arm_move()     → arm_control.cpp, ar_tf_arm_5.cpp, Grab.cpp, state_machine.cpp
navToGoal()    → ar_tf_arm_5.cpp, auto_charge.cpp, auto_bkhome.cpp, auto_nav_goal_4/5.cpp, state_machine.cpp
set_relmove()  → auto_charge.cpp, auto_nav_goal_4/5.cpp, state_machine.cpp
```
没有共享库，一处 bug 需改多处。

### 两个工作空间
`src/` 和 `src_task_1/` 各自有 build/devel，编译需分别执行。两个 `auto_bkhome_pkg` 一个空壳一个有实现，容易混淆。

### 共享库未引入
```
当前：每个 .cpp 自己实现 navToGoal/arm_move/set_relmove
推荐：
  robot_core/    → 共享库 (librobot_core.so)
  robot_apps/    → 功能节点 (链接共享库)
  robot_params/  → YAML 配置文件
```

## 优先建议

1. 将 `state_machine_pkg` 的残余硬编码（grasp 坐标、中转箱坐标、50mm 偏移）参数化
2. 统一两个工作空间为一个 catkin workspace
3. 将重复的核心函数抽取为共享库
