# ROS 项目硬编码问题分析

## 已改善的硬编码问题

### state_machine_pkg — 使用参数服务器
通过 `ros::param::param()` 加载配置，配合 `state_machine.launch` 集中管理参数：
```cpp
ros::param::param("~ar_id_1", ar_id_1_, 1);
ros::param::param("~pick_point_1_x", pick_point_1_x_, 0.5);
ros::param::param("~timeout_nav", timeout_nav_, 60.0);
```
**残余**: `grasp(107,185,42)` 的抓取坐标、`BOX_X=80` 的中转箱坐标、`z+50` 偏移量仍为字面量。

### Grab.cpp — 运行时放置点选择
```cpp
nh.param("placement_point_1_occupied", point1_occupied, false);
if (!point1_occupied) { /* 出刀点1 */ } else { /* 出刀点2 */ }
```

## 仍然存在的硬编码问题

### 坐标/位姿
`arm_control.cpp`、`ar_tf_arm_5.cpp`、`Grab.cpp` 中的导航点、机械臂坐标均为字面量，换场地需改源码。

### AR 标签 ID
```cpp
get_transform("Base", "ar_marker_1", ...);   // ar_tf_arm_5.cpp
set_ARtrack(0, 0.4);                          // 多个文件
```

### 坐标参考系
`"map"`、`"odom"`、`"Base"` 写死在代码中。

### 话题/服务名
`"/goto_position"`、`"/swiftpro/on"`、`"/track"` 等全部为字符串字面量。

### 机械臂偏移量
```cpp
// state_machine.cpp
srv.request.pose.position.z = z + 50;  // 50mm 字面量
// ar_tf_arm_5.cpp
void grab(x, y, z, off = 10) { ... }   // 10mm 默认值
```

### 区域编码 / 节点名
```cpp
setlocale(LC_CTYPE, "zh_CN.utf8");      // 所有文件
ros::init(argc, argv, "arm_control_node"); // 每个文件各自硬编码
```

### remove_back_pkg
```cpp
set_relmove(-0.18, 0, 0);               // 位移量硬编码
```
